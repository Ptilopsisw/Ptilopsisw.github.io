A 传球

原题ARC124E

一个比较难的DP

首先可以发现必定有一个人没有传球，因为每个人都少传一个球的话状态不变。

然后考虑式子的实际含义，即为每个人在自己手中选一个球的方案数。

所以我们可以对着实际含义来做，每个人手上的球的来源只有原来没有传出去的球和上一个人传过来的球，所以我们可以将球的来源记录在DP状态里。

设 fi,0f_{i,0}fi,0​ 表示 第 i−1i-1i−1 个人从自己手里选球时前 i−1i-1i−1 个人的选球方案数， fi,1f_{i,1}fi,1​ 表示 第 iii 个人从传过来的球中选球时前 iii 个人的选球方案数。(将两个状态错开一个是为了方便转移)

转移就分 444 中情况:

  1. 从 fi,0f_{i,0}fi,0​ 转移到 fi+1,0f_{i+1,0}fi+1,0​ : 自己和下一个人都不选被传的球，所以剩下几个球就有几种方案，方案数 ∑i=1aii\sum\limits _{i=1}^{a_i} ii=1∑ai​​i 。

  2. 从 fi,0f_{i,0}fi,0​ 转移到 fi+1,1f_{i+1,1}fi+1,1​ : 有两个人没有计算，且都在 aia_iai​ 中选球：∑i=1ai−1i(ai−i)\sum\limits _{i=1}^{a_i-1} i(a_i-i)i=1∑ai​−1​i(ai​−i)

  3. 从 fi,0f_{i,0}fi,0​ 转移到 fi+1,0f_{i+1,0}fi+1,0​ : 需要计算 aia_iai​ 中传出多少球，方案数 ai+1a_i+1ai​+1

  4. 从 fi,1f_{i,1}fi,1​ 转移到 fi+1,1f_{i+1,1}fi+1,1​ : 和第一个类似，方案数 ∑i=1aii\sum\limits _{i=1} ^{a_i} ii=1∑ai​​i 。

因为是一个环，需要DP一圈后统计 111 处的答案，可以枚举 111 的决策。(代码中由 A 实现 111 的决策)

但是这样并没有保证传球数最小为 000，可以容斥一下，算出所有的再减去 最小传球数为 111 的就行了(代码中由 B 保证)

C 龙与地下城

原题洛谷P8434

这是一道结论题： 所有子串的装饰子集与原集合的装饰自己相同。

证明分两步：

  1. 原序列装饰子集的元素一定在子串的装饰子集中： 对于一个元素 a∈Aa \in Aa∈A，aaa 一定被划分在某个子串中，整个序列中没有二进制位包含 aaa 的元素， 子串中也不可能存在，所以 aaa 一定属于它所在子串的装饰子集，而题目要求所有子串的装饰子集都相同，所以 aaa 一定在所有子串的装饰子集中。

  2. 不在原序列装饰子集的元素一定不在子串的装饰子集中: 若 a∉Aa \not\in Aa∈A，则一定存在 b∈Ab \in Ab∈A 使得 a∣b=ba|b = ba∣b=b , 而 bbb 一定在子串的装饰子集中，所以 aaa 一定不在子串的装饰子集中。

所以我们可以先处理出原序列的装饰子集(dfs即可)，然后用双指针处理出在每一个右端点 iii 最近的 lil_ili​ 使得 [li,i][l_i,i][li​,i] 包含整个装饰子集。

最后利用 lil_ili​ 跑一个简单的DP即可解决，DP可以用前缀和优化。

低仿机器人(robo)

一个纯模拟题，没什么好说的。

有一些坑点需要注意:

  1. 不要出现 END 或 ERROR 后就退出循环，要把当前数据点的输入全读取完再跑下一个数据点。

  2. 指令中可能存在小数，判一下是否有小数点就行了。

  3. 机器人不会执行发生 ERROR 的那条指令，需要输出发出这条指令前的状态.

迷路的刺豚(expand)

因为 p≤15p \le 15p≤15, 可以考虑状压DP。

设 fi,jf_{i,j}fi,j​ 表示 经过的菜店集合为 iii，当前位置在第 jjj 个菜店时的最短路，gi,jg_{i,j}gi,j​ 表示最大体格之和。

转移非常好写，只需要预处理出一些东西就行了。

首先需要预处理出在每个格子的最大体格，可以将障碍和边界看作起点，进行多起点的八连通bfs即可。

然后需要处理菜店之间两两的最短路和最大体格之和，可以跑 ppp 遍 Dijkstra 处理出来。

生日(birthday)

又是一道结论题：当区间长度 len≥14len \ge 14len≥14 时答案一定为 Yes。

证明: 对于一个元素数量为 lenlenlen 的集合，选取子集的方案数为 2len2^{len}2len，而选取元素总和的值域为 [1,v∗len][1,v*len][1,v∗len]，所以当 2len≥v∗len2^{len} \ge v*len2len≥v∗len 时，一定有解，此时 len≥14len \ge 14len≥14。

对于 len<14len < 14len<14 的询问，可以折半搜索解决，每个数的系数可能为 1,−1,01,-1,01,−1,0，当总和为 000 时代表有解。

有一个小技巧: 每次清空桶时可以不用 memset ，可以将桶的类型改为 int, 然后记录一个时间戳，就可以做到 O(1)O(1)O(1) 清空桶了。

关于修改操作，因为是模 vvv 意义下，肯定存在循环，所以我们可以倍增搞出每个数进行 2i2^i2i 次操作后的值，因为只需要计算 len≤14len \le 14len≤14 时的答案，所以每次暴力把数都算出来就行了，区间修改可以写个线段树来维护每个数修改了多少次。

bridge

这题题意有一些不清楚，题目问的是： 从一个特定点开始，可以左右走，最后回到这个点，且每两次经过这个点的时间间隔不超过 mmm 。

我们可以用DP搞，设 fif_{i}fi​ 表示走 iii 步回到起点且满足时间限制的方案数。

因为向左走和向右走是对称的，所以只计算其中一种然后乘 222 即可。

这时我们需要求出走 2x2x2x 步回到起点的方案数，且中途不能回到起点。(防止重复)

这个中途不能回到起点比较难计算，我们可以先走一步离开起点，然后计算可以回到起点的方案数，再走一步回来的方案数，将其设为 d2xd_{2x}d2x​

所以转移方程就很好写了: fi=∑j=2mfi−j∗dj−2∗2f_i = \sum_{j=2}^{m} f_{i-j}*d_{j-2}*2fi​=∑j=2m​fi−j​∗dj−2​∗2

因为这是个常系数转移方程，可以矩阵加速递推优化。
