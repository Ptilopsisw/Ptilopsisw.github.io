精神焕发

Description

给出一棵 nnn 个点的树，点从 111 至 nnn 编号。每个点 uuu 上有足够多瓶加血 wuw_uwu​ 的药水，如果原来你的血量为 xxx*，那么喝下这瓶药水后你的血量会变为 x+wux+w_ux+wu​。

有 qqq 次询问。每次询问给出数 xxx 与两个点的编号 sss、ttt。

这次询问中，你的初始血量为 xxx，一开始在 sss，想要走到 ttt。每次走到一个点，你就会喝下这个点上的药水（一开始来到点 sss 时会喝下 sss 上的药水）。你可以多次经过一个点，每次经过时都会喝下一瓶这个点上的药水。如果某个时刻你的血量小于 000，那么你就死了。你想知道你是否可以在不死的情况下从 sss 走到 ttt。

更形式化地，你需要判断是否存在一个点的编号组成的序列 p1…kp_{1 \dots k}p1…k​，满足：

  • p1=sp_1=sp1​=s，pk=tp_k=tpk​=t；
  • 对于任意 i∈[1,k−1]i \in [1,k-1]i∈[1,k−1]，树上的点 pip_ipi​ 与 pi+1p_{i+1}pi+1​ 之间有一条边。
  • 对于任意 i∈[1,k]i \in [1,k]i∈[1,k]，有 x+∑j=1iwpj≥0x+\sum_{j=1}^i w_{p_j} \ge 0x+∑j=1i​wpj​​≥0。

ppp 的长度 kkk 可以由你决定，并且 ppp 中可以存在相同的元素。

Input

输入的第一行包含两个数 nnn、qqq。

第二行包含 nnn 个数 w1..nw_{1..n}w1..n​。

接下来 n−1n-1n−1 行，每行包含两个数 uuu、vvv，表示树中存在连接点 uuu 与点 vvv 的一条边。

接下来 qqq 行，每行包含三个数 xxx、sss、ttt，表示一组询问，保证 s≠ts\neq ts=t。

Output

输出应包含 qqq 行，第 iii 行包含一个字符串 Yes 或 No，表示第 iii 次询问的答案。

Hint

对于 100%100\%100% 的数据，1≤n,q≤1061\le n, q\le 10^61≤n,q≤106，∣wi∣≤109|w_i|\le10^9∣wi​∣≤109，0≤x≤10180\le x\le 10^{18}0≤x≤1018。

Solution

从 sss 到 ttt 顶多只有两种方案：要么直接 rua 过去，要么找一个能反复横跳的地方无限加血。

无限加血的地方只能是一条两端点权值和 >0>0>0 的边。

如果到达这样的边的任意一个端点，那么就可以获得无限血，把这样的点标记为无限血点。

我们可以通过树形 DP 求出每个点到无限血点最少需要多少初始血量。

直接 rua 过去需要判断树上路径的前缀和最小值，容易发现前缀和最小值是整个路径或者是整个路径除去 ttt 点，否则路径上一定会有一个无限血点，所以这一部分直接判路径和就行。

时间复杂度取决于 LCA, 最优可以做到 O(n+k)O(n+k)O(n+k)。

串串

Description

FAT 想要烫串串。

现在有一个长度为 nnn 的字符串 SSS 与一个长度为 mmm 的字符串 TTT。

定义字符串 SiS_iSi​ 表示将从 SSS 的第 iii 个字符后断开得到 AAA，BBB，将 BBB 从尾到头形成的字符串设为 B′B'B′，将 B′B'B′ 接在 AAA 后面得到的字符串。

对于每个 iii，FAT 想知道 TTT 在 SiS_iSi​ 中出现的次数。

Input

第一行包含一个字符串 SSS。

第二行包含一个字符串 TTT。

Output

输出 nnn 行每行一个整数，第 iii 行的数表示 TTT 在 SiS_iSi​ 中出现的次数。

Hint

对于 100%100\%100% 的数据，1≤m≤n≤2×1061\le m \le n\le 2\times 10^61≤m≤n≤2×106。

Solution

TTT 在 Si=AB′Si=AB′Si=AB′ 中的出现次数是 TTT 在 AAA 中的出现次数 + TTT 在 B′B'B′ 中的出现次数 + TTT 跨过 AAA 和 B′B′B′ 的出现次数。

前两个很好处理，考虑最后一个。

TTT 的一个后缀要与 B′B′B′ 的前缀匹配，剩余的前缀要与 AAA 的后缀匹配。

可以先处理出所有跟 B′B′B′ 的前缀匹配的 TTT 后缀，然后询问它们剩的前缀有多少个是 AAA 的后缀。

可以建出 KMP 的 fail tree 解决。

时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。
