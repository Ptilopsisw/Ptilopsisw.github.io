owo

Description

有两组玩家进行游戏，第一组玩家有 nnn 个人，第二组玩家有 mmm 个人，两组之间玩家两两会进行比赛，一共进行 n∗mn*mn∗m 场比赛。

第一组玩家能力值为 a1,a2,⋯ ,ana_1,a_2,\cdots,a_na1​,a2​,⋯,an​, 第二组玩家能力值为 b1,b2,⋯ ,bmb_1,b_2,\cdots,b_mb1​,b2​,⋯,bm​ ，其中能力值为 111 到 222 之间的浮点数，两个能力值为 aaa 和 bbb 的玩家进行比赛的时候，能力值为 aaa 的玩家获胜概率为 a/(a+b)a/(a+b)a/(a+b)。

请求出所有比赛结束后，第一组每个玩家获胜场次的期望。

Input

第一行两个整数 n,mn, mn,m。

接下来一行包含 nnn 个实数 a1,a2,⋯ ,ana_1, a_2, \cdots, a_na1​,a2​,⋯,an​ 。

最后一行包含 mmm 个实数 b1,b2,⋯ ,bmb_1, b_2, \cdots, b_mb1​,b2​,⋯,bm​。

输入的实数小数位数不超过 888 位。

Output

输出 nnn 行，每行一个实数表示每个玩家获胜场次的期望，要求误差在 10310^3103 一下。

Hint

n,m≤106n,m \le 10^6n,m≤106

Solution

因为答案精度仅需 10310^3103 ，我们可以发挥人类智慧，将每个输入乘以 10410^4104，然后舍弃剩余小数位，这样所有的输入就变成值域在 10410^4104 以内的整数了，然后用 O(值域2)O(值域^2)O(值域2) 做法草过去就行。

qaq

Description

有个无穷大的 kkk 维空间，一开始在 (0,0,⋯ ,0)(0,0,\cdots,0)(0,0,⋯,0) ，每次等概率地在 2k2k2k 个方向中随机选一个方向走一步。

求走 nnn 步之后能经过的不同位置的个数的期望，输出期望乘上 (2k)n(2k)^n(2k)n 后对 modmodmod 取模的结果。

Input

一行三个正整数 n,k,modn, k, modn,k,mod

Output

一行一个整数表示答案。

Hint

n≤2000,k≤10,mod≤109+7n \le 2000, k \le 10, mod \le 10^9+7n≤2000,k≤10,mod≤109+7

Solution

fif_ifi​ 表示 iii 步之后第一次经过这个格子的方案。

gig_igi​ 表示 iii 步之后停在这个格子的方案。

hih_ihi​ 表示从一个位置出发到原点的方案。

有 fi=gi−∑j+k=ifj∗hkf_i = g_i - \sum\limits _{j+k=i} f_j*h_kfi​=gi​−j+k=i∑​fj​∗hk​

考虑 hih_ihi​ 与哪个格子没有关系，式子是一个 g→fg \rightarrow fg→f 的线性变换。

对所有格子求和，有 dpi=(2k)i−∑j+k=idpj∗hkdp_i = (2k)^i - \sum\limits _{j+k=i} dp_j*h_kdpi​=(2k)i−j+k=i∑​dpj​∗hk​ 。

用组合数暴力求卷积即可。

考虑求 hih_ihi​，发现每一维相互独立，直接背包就行。

tvt

Description

给一棵树，边带权，qqq 次询问，每次询问 Alice 在 t1t1t1 时刻从 x1x1x1 出发走到 y1y1y1, Bob 在 t2t2t2 时刻从 x2x2x2 出发走到 y2y2y2 ，速度相同，问两人是否在某一时刻出现在同一条边上(不包括顶点)。

Input

第一行两个整数 n,qn, qn,q。

接下来 n−1n-1n−1 行，每行三个整数 x,y,wx, y, wx,y,w ，代表树上一条边。

接下来 qqq 行，每行六个整数 x1,y1,t1,x2,y2,t2x1, y1, t1, x2, y2, t2x1,y1,t1,x2,y2,t2。

Output

输出 qqq 行，每行为 YES 或 NO, 表示两个人是否在某一时刻同时出现在一条边上。

Hint

n,q≤105n,q \le 10^5n,q≤105

Solution

先求出路径交，然后分情况讨论。

如果两个人同向，那么比较路径交上最长边和时间差即可。

如果两个人相向，那么计算出相遇点，然后倍增或树剖判断相遇点是否恰好为一个顶点即可。
