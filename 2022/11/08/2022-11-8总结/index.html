环

Description

对于一个长度为 nnn 的 010101 串，下标从 000 到 n−1n-1n−1 。定义两种类型的操作：

A类型：选择一个 xxx，将序列循环右移 xxx 位，也就是新序列的第 (i+x) mod n(i+x) \bmod n(i+x)modn 位对应原序列的第 iii 位。

B类型：选择一个 xxx ，满足序列的第 xxx 个位置位 111 ，且第 (x+1) mod n(x+1) \bmod n(x+1)modn 位不为 111 ，交换序列的第 xxx 个位置和第 (x+1) mod n(x+1) \bmod n(x+1)modn 个位置的字符。

给定 n,k,ln,k,ln,k,l 。请构造一个由 lll 个长度为 nnn 的 010101 串构成的序列 SSS，下标从 000 开始。使得序列中每个串恰好有 kkk 个 111 。且对于 0≤i<l−10 \le i < l-10≤i<l−1，SiS_iSi​ 既可以通过一次A类型的操作，也可以通过一次B类型的操作，变为 Si+1S_{i+1}Si+1​。

Input

第一行一个整数 TTT 表示数据组数。

接下来 TTT 行，每行三个整数 n,k,ln,k,ln,k,l。

Output

对于每组数据，如果无解输出"NO",有解输出一行"YES"，并输出 lll 行描述序列 SSS。多解任意输出一组。

Hint

2≤n,l≤1002 \le n,l \le 1002≤n,l≤100, 0≤k≤n0 \le k \le n0≤k≤n, 1≤T≤101 \le T \le 101≤T≤10。

Solution

首先，容易证明当 kkk 与 nnn 不互质时无解。 设 SiS_iSi​ 中所有 111 的位置的下标之和为 WiW_iWi​ ，假设 SiS_iSi​ 能够通过循环移位得到 Si+1S_{i+1}Si+1​ ，那么必然存在整数 xxx ，使得 Wi+xk≡Wi+1(modn)W_i+xk \equiv W_{i+1} \pmod nWi​+xk≡Wi+1​(modn) 。同时如果 SiS_iSi​ 能够通过将一个 111 右移一位得到 ，那么有 Wi+1≡Wi+1(modn)W_i+1 \equiv W_{i+1} \pmod nWi​+1≡Wi+1​(modn) ，由两式可以得到 xk≡1(modn)xk \equiv 1 \pmod nxk≡1(modn) ，所以 n,kn, kn,k 互质是解存在的必要条件。

当 kkk 与 nnn 互质时，容易给出一种构造。 因为 kkk 与 nnn 互质，所以 kkk 一定存在模 nnn 意义下的逆元，令 S0S_0S0​ 的 0k,1k,⋯ ,k−1k\frac{0}{k}, \frac{1}{k}, \cdots, \frac{k-1}{k}k0​,k1​,⋯,kk−1​ 位置处（这里和后面的位置均是指在模 nnn 意义下）为 111 ，其余处为 000 。同样的对于 SiS_iSi​ ，我们令其 ik,i+1k,⋯ ,k+i−1k\frac{i}{k}, \frac{i+1}{k}, \cdots, \frac{k+i-1}{k}ki​,ki+1​,⋯,kk+i−1​ 处为 111 ，其余处为 000 。显然，从 SiS_iSi​ 变成 Si+1S_{i+1}Si+1​ 可以通过向右循环移位 1k\frac{1}{k}k1​ 实现，同时发现 ik+1=k+(i+1)−1k\frac{i}{k} + 1 = \frac{k+(i+1)-1}{k}ki​+1=kk+(i+1)−1​ ，于是我们将 SiS_iSi​ 的 ik\frac{i}{k}ki​ 位置变成 000 ，将 ik+1\frac{i}{k}+1ki​+1 位置变成 111，即可用 bbb 操作将 SiS_iSi​ 变成 Si+1S_{i+1}Si+1​ 。

DNA序列

Description

给定 nnn 个只包含 “A”,“T”,“C”,“G” 字符串，每个字符串取一个非空前缀，然后将这些前缀按任意顺序链接起来，问能获得的字典序最小的字符串是什么。

Input

第一行一个整数 nnn，表示字符串个数。

接下来 nnn 行，每行一个字符串。

Output

输出一行一个字符串。

Hint

1≤n,m≤501 \le n, m \le 501≤n,m≤50。

Solution

首先，每个字符串后面加一个很大的字符（例如"Z"）。

对于每个字符串 sss ，找到最短的前缀 xxx 满足 x∞<sx^\infty < sx∞<s ，然后将字符串 sss 分成 xy+zx^y+zxy+z ，使得 xxx 不是 zzz 的前缀。

那么合并的顺序一定是按照 x∞x^\inftyx∞ ，字典序升序然后相同按照 z+′Z′z+'Z'z+′Z′ 的字典序降序。

我们可以考虑是每次我们希望找个最小的 xxx ，然后加入尽量多的 xxx，但是 xxx 加完之后我们希望能加的字符尽量小，所以我们把尾巴字典序最小的放后面。

知道顺序之后，倒着枚举前缀的长度暴力贪心就行了。

探寻

Description

给定一棵 nnn 个点， n−1n-1n−1 条边的树，边带权，节点编号为 0∼n−10 \sim n-10∼n−1 ，其中根的编号为 000，一开始你在根上，第一次走一条边需要花与边权相同数目的钱，走过一次这条边后再走这条边就不需要花钱，如果钱不够则不能走，第一次走到一个点 xxx 可以获得 valxval_xvalx​ 的钱，之后再到达 xxx 就不会获得钱，标记其中一个叶子为目标节点，求出到达目标节点所需的最少的初始钱数。

Input

第一行一个整数 nnn。

接下来 n−1n-1n−1 行，每行三个整数 pi,costi,valip_i, cost_i, val_ipi​,costi​,vali​，表示节点 iii 的父节点为 pip_ipi​，与父节点所连的边的边权为 costicost_icosti​，第一次走到节点 iii 可以获得 valival_ivali​ 的钱。

仅有一个叶子节点满足 vali=−1val_i = -1vali​=−1 ，该节点为目标节点。

Output

一个整数表示最少的初始钱数。

Hint

2≤n≤2×105,0≤costi,vali≤1092 \le n \le 2 \times 10^5, 0 \le cost_i, val_i \le 10^92≤n≤2×105,0≤costi​,vali​≤109。

Solution

每次只走一步的贪心显然是错的，因为没有考虑走完这一步后的情况，所以我们从 xxx 走到 yyy 的时候，需要考虑 yyy 整棵子树内的情况。

我们将走一个子树内的连通块成为一个决策，一个决策有两个属性 (need,get)(need, get)(need,get)，代表使用这个决策的前提是有 needneedneed 的钱，使用完这个决策后的收益是 getgetget，只有 get>0get > 0get>0 的决策才是有用决策。

设当前节点为 xxx ，y∈sonxy \in son_xy∈sonx​ ，只走 xxx 这一个点的决策显然是 (costx,valx−costx)(cost_x, val_x-cost_x)(costx​,valx​−costx​)，如果 valx−costx<0val_x -cost_x < 0valx​−costx​<0 ，那么就需要从 yyy 中选择决策增加 xxx 的收益，这个可以将决策按 needneedneed 比较，用可并堆维护子树内的决策即可，时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。
