最短子列

Description

对于一个字符串 S=S1S2…SnS=S_1S_2\dots S_nS=S1​S2​…Sn​ ，定义它的一个子序列是任意满足 1≤i1<i2<⋯<ik≤n1 \le i_1 < i_2 < \dots < i_k \le n1≤i1​<i2​<⋯<ik​≤n 的字符串 Si1Si2…SikS_{i_1}S_{i_2} \dots S_{i_k}Si1​​Si2​​…Sik​​ 。字符串 SSS 的非子序列即不是 SSS 的子序列的字符串。

对于两个字符串 S,TS,TS,T ，我们定义它们的最短公共非子序列为一个最短的字符串，使得它既是 SSS 的非子序列，也是 TTT 的非子序列。

这里 S,YS,YS,Y 仅由 000 和 111 构成。现在请你求出字典序最小的最短公共非子序列。

Input

第一行两个正整数 n,mn, mn,m ，表示两个字符串 S,TS,TS,T 的长度。

第二行一个长度为 nnn 的、仅由 0,10,10,1 组成的字符串 SSS ，表示第一个字符串。

第三行一个长度为 mmm 的、仅由 0,10,10,1 组成的字符串 TTT ，表示第二个字符串。

Output

输出一行一个字符串，表示字典序最小的最短公共非子序列。

HINT

1≤n,m≤40001 \le n,m \le 40001≤n,m≤4000

Solution

考虑子序列的匹配过程，设当前子序列匹配到 iii, 若添加一个字符，则下一步应匹配到 iii 后面的第一个这个字符，如果没有则代表是非子序列。

然后就可以 DP 了，然而我直接建边跑了个 bfs, 本质上是一样的.

树上深搜

Description

有份 DFS 的代码：

这个代码表示对一个点数为 nnn 的无向连通图进行遍历。

其中 g[][] 是一个布尔数组，如果图有边 (x,y)(x,y)(x,y) 则 g[x][y]=g[y][x]=true，否则 g[x][y]=g[y][x]=false 。特别地，对于任意的 1≤x≤n1 \le x \le n1≤x≤n 都有 g[x][x]=false。

link(x,y) 表示在另一个点数为 nnn 的图 TTT 上连边 。注意，初始时 TTT 中只有 nnn 个点而没有任何边。容易得到，执行 dfs(1) 之后 TTT 将会是一棵树。

求对于给定的树 TTT ，有多少个没有重边和自环的无向连通图 GGG 满足对 GGG 执行 dfs(1) 之后得到的树 与给定的树完全一样？

两个图完全一样，当且仅当这两个图的节点数相同并且对于第一个图的任意一条边 (x,y)(x,y)(x,y) ，第二个图中都有边 (x,y)(x,y)(x,y) 存在。

Input

第一行一个正整数 nnn ，表示树 TTT 的点数，也是无向连通图的点数。

接下来的 n−1n-1n−1 行，每行两个正整数 x,yx,yx,y ，表示树 TTT 上有边 (x,y)(x,y)(x,y) 。

Output

输出一个整数表示答案。由于答案可能很大，你只要输出其对 109+710^9+7109+7 取模后的结果。

HINT

n≤2×105n \le 2 \times 10^5n≤2×105

Solution

考虑哪些边连不连无影响，对于一个节点 xxx，如果 xxx 的祖先中有比 xxx 编号小的节点 yyy ，则 (x,fay)(x,fa_y)(x,fay​) 这条边可连可不连，问题就变成了统计一个点的祖先中有多少点比它编号小，因为 111 号点没有父节点所以 111 不能算上，这个可以 dfs 时维护一个 bit 实现。

五彩斑斓

Description

小 L 想要粉刷一块 n×mn \times mn×m 的木板，现在他手上有 ccc 桶不同颜色的油漆。

小 L 是一个很怕麻烦的人，他每次只会粉刷完整的一行或一列，并且他希望粉刷的次数尽量少。

现在给你小 L 的粉刷方案，即粉刷完成之后，每个小方格应该是什么颜色的。

请你给出一个粉刷方案，使得粉刷次数最少。

当然，也可能不存在任何合法方案，这时请你告诉小 L 他的设计不合法。

Input

输入的第一行包含三个整数 n,m,cn, m, cn,m,c ，含义如题所示。

接下来的 nnn 行，每行包含 mmm 个整数。令第 iii 行的第 jjj 个数字为 coli,jcol_{i,j}coli,j​ ，满足 0≤coli,j≤c0 \le col_{i,j} \le c0≤coli,j​≤c 。如果 coli,j=0col_{i,j}=0coli,j​=0 ，则说明格子 (i,j)(i,j)(i,j) 不应该被粉刷，否则说明格子 (i,j)(i,j)(i,j) 应该被粉刷成颜色 coli,jcol_{i,j}coli,j​ 。

Output

如果不存在合法方案，仅输出一行，包含一个整数 −1-1−1。

否则，输出的第一行包含一个整数 ttt ，表示最少的粉刷次数。

接下来的 ttt 行，按照顺序输出 ttt 次粉刷操作。第 iii 行描述第 iii 次操作，包括空格隔开的一个字符 optiopt_iopti​ 以及两个数字 numinum_inumi​ 和 colicol_icoli​ 。

  • optiopt_iopti​ 为 R 或 C，分别代表粉刷一行或者一列 。
  • numinum_inumi​ 为粉刷的列或行的编号。
  • colicol_icoli​ 为粉刷的颜色。

如果存在多种合法且操作次数最少的方案，输出任意一种即可。

HINT

1≤n,m≤50, c≤1001 \le n,m \le 50, \, c\le 1001≤n,m≤50,c≤100

评分程序将对选手输出按照如下标准评分：

  1. 如果不存在输出文件，或者输出不符合规范，得 分。
  2. 如果最少的粉刷次数不正确，且方案不合法，得 分。
  3. 如果最少的粉刷次数不正确，但方案合法，可以获得 的分数。
  4. 如果最少的粉刷次数正确，但方案不合法，可以获得 的分数。
  5. 如果最少的粉刷次数正确，且方案合法，可以获得 的分数。

这里方案合法是指方案行数等于 ， 且按照方案粉刷可以得到给定的设计。

注意获得 的分数的条件是方案格式正确，即按要求输出 行。当然这 个操作可以完全乱来，不过还是得符合操作的格式。

Solution

首先粉刷次数的上界是 n+m−1n+m-1n+m−1，因为如果刷了 n+mn+mn+m 次的话，第一次刷的颜色会全部被覆盖掉，那么第一次的操作就没用了。

这样的话我们知道至少有一行或者一列没有粉刷颜色，那么我们就枚举没有刷颜色的这一行 (列的话把矩阵转置一下就行)，因为这一行没有刷颜色，所以可以通过这一行唯一确定每一列刷的颜色，然后就可以确定一些行刷的颜色和刷的顺序了(确定不了的视为没刷颜色就行)。方案的话用拓扑排序处理一遍就行了。
