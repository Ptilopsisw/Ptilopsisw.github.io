取石子

Description

有 nnn 堆石子，第 iii 堆石子有 xix_ixi​ 个。 Alice 和 Bob 轮流取石子， Alice 先手。每人每次从一堆石子中取 a∼ba\sim ba∼b 个。无法操作的人失败。

此外，当一个人取完一堆石子时他会立即获胜。求谁会获胜。有多组数据。

Input

第一行一个整数 ttt 表示数据组数。接下来每组数据两行：第一行三个整数 n,a,bn,a,bn,a,b ，第二行 nnn 个整数 x1,x2,...,xnx_1,x_2,...,x_nx1​,x2​,...,xn​ 。

Output

每组数据输出一行一个字符串表示答案。

solution

考虑 SG 定理，先只看一堆石子的情况，石子个数为 1∼a−11 \sim a-11∼a−1 时，那么先手无法操作，后手必胜，所以 SG 值为 000，对于 a∼ba \sim ba∼b 的情况，因为这是必胜态的终止局面，所以没有 SG 值，可以看作 −1-1−1 或者 +∞+ \infin+∞ (我也不确定是不是没有 SG 值，但是这么算就是对的，我也不太会博弈论（逃)。因为每次取的石子数的限制是固定的，所以 SG 值肯定会出现循环节，暴力算 SG 打个表找规律就很容易发现 SG 值的循环节是 a+ba+ba+b 了。

整个游戏的 SG 值是所有子游戏 SG 值的异或和。

Extending Set of Points

Description

对于一个二维点的集合 SSS，对其不断进行以下操作得到的集合称为 SSS 的扩展集合 E(S)E(S)E(S)：一开始 E(S)E(S)E(S) 为 SSS。不断取四个数 x1,y1,x2,y2x_1,y_1,x_2,y_2x1​,y1​,x2​,y2​，使得 E(S)E(S)E(S) 中包含点 (x1,y1),(x1,y2),(x2,y1)(x_1,y_1),(x_1,y_2),(x_2,y_1)(x1​,y1​),(x1​,y2​),(x2​,y1​) 但不包含点 (x2,y2)(x_2,y_2)(x2​,y2​)，然后将点 (x2,y2)(x_2,y_2)(x2​,y2​) 加入 E(S)E(S)。重复这样的操作直到不能继续操作（即不存在符合要求的 x1,y1,x2,y2x_1,y_1,x_2,y_2x1​,y1​,x2​,y2​）为止。

一开始 SSS 为空，依次给出 qqq 个点 p1…qp_{1 \dots q}p1…q​，依次对于 1≤i≤q1 \le i \le q1≤i≤q，如果 pip_ipi​ 不存在于 SSS 中，那么将 p_ip**i 加入 SSS，否则将 pip_ipi​ 从 SSS 中删除。

请在每次加入或删除操作后，求出 E(S)E(S)E(S) 的大小。注意答案可能会超出 323232 位有符号整数的存储范围。

Input

输入第一行包含一个数 qqq，含义如上。

接下来 qqq 行，其中其 iii 行包含两个数 xi,yix_i,y_ixi​,yi​，表示点 pip_ipi​ 为 (xi,yi)(x_i,y_i)(xi​,yi​)。

Output

输出一行，包含 qqq 个数，第 iii 个表示在第 iii 次加入或删除操作（即处理完 pip_ipi​）后，你求出的 E(S)E(S)E(S) 的大小。

HINT

1≤q≤3×1051 \le q \le 3 \times 10^51≤q≤3×105，对于 1≤i≤q1 \le i \le q1≤i≤q 有 1≤xi,yi≤q1 \le x_i,y_i \le q1≤xi​,yi​≤q，输入的所有数为正整数。

solution

这个操作会将每个缺一个角的矩形添上那个角，如果我们将 xxx 相同或 yyy 相同的点连在一起，那么这个操作会将每个连通块补成一张网格，每个联通块点的贡献即为 xxx 坐标的个数乘以 yyy 坐标的个数。

这可以启示我们用并查集维护 x,yx,yx,y 坐标，我们可以记录每个集合中 x,yx,yx,y 坐标的个数，将一个坐标为 (x,y)(x,y)(x,y) 的点看成合并 x,yx,yx,y 这两个集合。

然后题目要求求出所有时间的答案，直接线段树分治即可。
